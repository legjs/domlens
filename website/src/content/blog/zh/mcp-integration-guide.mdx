## 什么是 MCP？

**模型上下文协议（MCP）** 是一个开放标准，使 AI 工具能够与外部服务通信。DOM Context 实现了 MCP 服务器，让 AI 编程助手可以直接访问实时 DOM 上下文。

## 前提条件

- Node.js 18 或更高版本
- 已安装 Claude Code CLI
- 已安装 DOM Context Chrome 扩展

## 第一步：启动运行时服务器

```bash
git clone https://github.com/user/claude-dom-context.git
cd claude-dom-context
pnpm install
pnpm runtime:start
```

服务器将自动发现 4777-4787 范围内的可用端口。

## 第二步：配置 Claude Code

将以下内容添加到 Claude Code 的 MCP 配置中：

```json
{
  "mcpServers": {
    "dom-context": {
      "command": "node",
      "args": ["src/server/index.ts"],
      "type": "stdio"
    }
  }
}
```

## 第三步：在工作流中使用

1. 在 Chrome 中打开任意网页
2. 激活 DOM Context 检查器（Alt + 悬停）
3. 单击或 Ctrl+单击元素捕获上下文
4. 在 Claude Code 中，AI 现在可以调用 `get_latest_context` 获取捕获的 DOM 数据
5. 使用 `get_prompt` 获取即用的 Markdown 描述

## 可用工具

| 工具 | 描述 |
|------|------|
| `get_latest_context` | 获取最新的 DOM 上下文捕获 |
| `get_prompt` | 获取格式化的 AI 对话 Markdown 提示 |
| `list_contexts` | 列出所有存储的上下文历史 |
| `clear_contexts` | 清除所有存储的上下文数据 |
| `get_user_prompt` | 获取浮动面板的最新提示 |

## 使用技巧

- 使用**多元素选择**（Ctrl+单击）捕获多个相关元素
- **上下文压缩**引擎会自动减少冗余的 DOM 数据
- 使用**浮动面板**在元素数据旁边添加自定义备注
- 上下文历史默认存储最多 20 条记录

## 故障排除

### 服务器无法启动
确保端口 4777-4787 未被其他应用程序占用。

### Claude Code 无法连接
验证 MCP 配置中的路径是否正确指向 `src/server/index.ts` 文件。

### 上下文为空
确保检查器激活后，您已通过悬停或单击捕获了至少一个元素。
