# AI Runtime Inspector（MVP）技术方案

## 项目定位

不是 IDE。

而是：

```text
给 AI Coding Agent 提供高质量网页运行态上下文
```

核心能力：

```text
用户点击网页元素
↓
自动分析运行态结构
↓
生成 AI 可理解上下文
↓
发送给 Claude / Cursor / OpenAI Agent
```

目标：

解决 AI Coding 最大痛点：

```text
AI 只能看源码
无法理解运行态布局
```

---

# 一、MVP 功能目标

第一阶段只做：

## MVP 1

```text
点击元素
→ 自动生成 AI Context
→ 一键复制
```

不做：

* 自动修复
* 自动改代码
* IDE
* 多人协作
* 云端同步
* 截图分析

# 模块3：Layout Chain Analyzer（重点）

这是整个产品核心竞争力之一。

---

# 为什么重要？

真实 UI 问题：

```text
80% 都来自父级布局约束
```

例如：

* flex-shrink
* overflow hidden
* absolute parent
* transform
* stacking context

---

# 实现逻辑

递归：

```ts
el.parentElement
```

直到：

```text
document.body
```

---

# 只保留“布局关键节点”

例如：

```ts
display:flex
display:grid
overflow:hidden
position:absolute
```

---

# 输出格式

```json
{
  "layoutChain": [
    {
      "tag": "div",
      "class": "toolbar",
      "display": "flex",
      "width": "320px"
    }
  ]
}
```

# 模块4：React Fiber Analyzer（极重要）

这是产品上限。

---

# 目标

实现：

```text
DOM
→ React Component
```

---

# 原理

React DOM 节点存在：

```ts
__reactFiber$
```

---

# 获取方式

```ts
const reactKey = Object.keys(el).find(
  key => key.startsWith("__reactFiber$")
)
```

---

# 获取 fiber

```ts
const fiber = el[reactKey]
```

---

# 提取：

```ts
{
  componentName,
  props,
  stateNode
}
```

---

# 获取组件名

```ts
fiber.type?.name
```

---

# 示例输出

```json
{
  "component": "LoginButton"
}
```
